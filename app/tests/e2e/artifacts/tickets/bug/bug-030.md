# bug-030: Player grid has no touch event support for mobile devices

## Summary

The player VTT grid (`GridCanvas.vue` in player mode) has zero touch event handlers. Mobile devices (the primary target audience for the player view) cannot pan or zoom the grid. The design spec P1 acceptance criteria states "Pinch-to-zoom works" but this is not implemented.

Additionally, in player mode, left-click panning is blocked because every left-click is intercepted as a cell/token click. Desktop users must use middle-click to pan, which many mice don't have.

## Affected Files

- `app/components/vtt/GridCanvas.vue` — lines 257-281 (player mode left-click interception)
- `app/composables/useGridInteraction.ts` — no touch event handlers

## Current Behavior

- Left-click in player mode always emits `playerCellClick` — no click-vs-drag detection
- No `touchstart`, `touchmove`, `touchend` handlers exist
- No pinch-to-zoom gesture recognition
- Mobile players see only the initially visible portion of the grid

## Expected Behavior

- Touch pan (single finger drag) should scroll the grid
- Pinch-to-zoom should work on mobile
- Left-click on desktop should distinguish between click (cell selection) and drag (panning)
- A tap on mobile should select a cell, a drag should pan

## Impact

Mobile players (phones/tablets) cannot navigate the tactical grid at all. This is a core usability blocker for the player view's primary use case (phone-based play during tabletop sessions).

## Suggested Fix

1. Add click-vs-drag detection: track `mousedown` position, if cursor moves >5px before `mouseup`, treat as pan instead of cell click
2. Add touch event handlers for pan (single finger) and pinch-to-zoom (two fingers)
3. Consider using a gesture library or implementing basic touch math (distance between two touch points for zoom factor)

**What was implemented:**

2. **Pinch-to-zoom:** Two-finger distance ratio scales zoom level, with pan offset adjusted to zoom toward pinch center
3. **Tap detection:** Touch without movement beyond threshold treated as tap (selects cell/token)
4. **Player mode:** onTouchTap callback routes taps to playerTokenSelect/playerCellClick events
5. **touch-action: none CSS:** Prevents browser default pan/zoom gestures
6. **preventDefault on all touch events:** Prevents synthesized click events (composable handles all touch taps directly)
