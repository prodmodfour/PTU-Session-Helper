# ux-011: Custom item form missing grantedCapabilities input field

## Summary

The custom item form in `HumanEquipmentTab.vue` (`confirmCustomItem`) builds an `EquippedItem` with DR, evasion, speed CS, and description fields, but does not expose a `grantedCapabilities` field. The API (equipment.put.ts Zod schema) accepts `grantedCapabilities: z.array(z.string().min(1)).optional()`, but GMs cannot specify capabilities through the custom item form.

This means homebrew equipment that should grant capabilities (e.g., "Desert Boots" granting Naturewalk (Desert)) cannot be created through the UI — only through direct API calls.

## Affected Files

- `app/components/character/tabs/HumanEquipmentTab.vue` (lines 304-318, `confirmCustomItem` method)

## Suggested Fix

Add a text input field (comma-separated or multi-chip) for `grantedCapabilities` to the custom item form modal. Parse the input into a string array before passing to the API.

## Impact

Low — catalog items already have grantedCapabilities. Only affects GMs creating homebrew equipment with capability grants.
