# UX-015: Replace alert() with inline UI for evolution prevention messages

## Summary

Both the Pokemon sheet page (`gm/pokemon/[id].vue`) and `XpDistributionResults.vue` use browser `alert()` to display the Everstone/Eviolite prevention message. The evolution check response already includes `preventedByItem` — this should be displayed inline in the UI rather than via a browser alert.

## Affected Files

- `app/pages/gm/pokemon/[id].vue` (line ~406)
- `app/components/encounter/XpDistributionResults.vue` (line ~237)

## Suggested Fix

Replace `alert()` calls with an inline notification or toast component showing "Evolution prevented by [Everstone/Eviolite]" near the evolution trigger point.

## Impact

Low — cosmetic UX improvement. `alert()` works but is inconsistent with the rest of the UI.
