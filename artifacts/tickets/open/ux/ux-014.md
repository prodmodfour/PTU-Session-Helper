# UX-014: Evolution undo snapshot staleness warning

## Summary

The evolution undo system (`useEvolutionUndo` composable) stores snapshots in `useState` with no expiry mechanism. The undo button remains available indefinitely within the same session. If the GM undoes an evolution much later — after the Pokemon has participated in combat or had other changes applied — the undo reverts ALL stats/moves/abilities to the pre-evolution snapshot, discarding any post-evolution modifications.

## Affected Files

- `app/composables/useEvolutionUndo.ts`

## Suggested Fix

Add a timestamp to each snapshot entry and either:
- Auto-clear after N minutes (e.g., 30 minutes)
- Warn the GM if the snapshot is stale ("This undo snapshot is X minutes old. Post-evolution changes may be lost.")
- Clear the snapshot when the Pokemon enters an encounter (the endpoint already rejects undo for Pokemon in active encounters, but the button still shows)

## Impact

Low — edge case where GM undoes evolution long after it occurred. Could cause unexpected data loss of post-evolution stat changes.
